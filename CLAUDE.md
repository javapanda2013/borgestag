# BorgesTag — Claude 作業ルール

## プロジェクト概要

Firefox WebExtension（Manifest V2）＋ Python Native Messaging バックエンド。
ローカル HTTP サーバーは存在しない。devサーバーの起動・プレビューパネルは使用しない。

---

## 開発作業ルール

### バージョン管理
- メジャー：基盤を大幅変更した場合
- マイナー：新機能追加
- パッチ：軽微な修正・機能調整・バグ修正

### 成果物チェックリスト（毎回必須）
1. `node --check` による構文チェック
2. `CHANGELOG.md` に変更内容を記載
3. `git commit & push`（リモート名：`borgestag`）
4. `python make_zip.py v{バージョン}` で Mozilla 申請用 ZIP を作成
5. ポートフォリオ更新（下記ルール参照）

### コーディング禁止事項
- 依頼と異なる修正・依頼範囲外の改善を無断で行わない
- 既存コードの構造を無断で大幅変更しない
- 日本語コメントを英語に変換しない
- 依頼範囲内のタスクを「別途ご依頼ください」として先送りしない

### バグ修正時のフロー（省略禁止）
1. 原因を調査・特定する
2. **修正方針をユーザーに提示し、確認を得てから実装に入る**
3. 実装後に構文チェック・バージョン更新・CHANGELOG 記載・コミット

口頭での「確認します」は記録されないため、必ずこのチェックリストに記載すること。

### ファイル編集時の注意
- テーブルを更新・行追加する際は、編集前にテーブル全体を Read して現在の状態を確認し、日付順・整合性を維持する（前セッションで他の行が追加・変更されている場合があるため）

### 外観・レイアウト関連の修正時（省略禁止）
修正に着手する前に、以下4点をすべて grep で確認すること：
1. JS ファイル内の CSS テンプレート文字列（`<style>` ブロック等）
2. 外部 CSS ファイル（`*.css`）
3. HTML ファイル内のインラインスタイル（`style="..."` 属性）
4. JS からの動的スタイル操作（`.style.maxHeight =` 等）

口頭での「確認します」は記録されないため、必ずこのチェックリストに記載すること。

### native/image_saver.py のバージョン管理（重要）
- `native/image_saver.py` を変更したリリースでは、ファイル先頭の `version:` コメント（2行目付近）を必ずそのリリースのバージョン番号に更新すること
- `manifest.json` のバージョン更新と同時に行う
- Python ファイルを変更しないリリース（拡張機能 JS のみの変更等）でも、バージョンコメントを合わせておくことを推奨する

## ポートフォリオ更新ルール

### 更新前に必ず行うこと（省略禁止）
1. ポートフォリオ全ファイルを製品名（`BorgesTag` など）で grep し、関連ページを網羅的に特定する
2. 特定したファイルと変更内容の一覧をユーザーに提示する
3. ユーザーの承認を得てから編集に入る

### 対象ファイルの例（BorgesTag 関連）
- `main/home.html` — 更新履歴テーブル
- `main/ai.html` — 成果物カード（バージョンタグ・説明文・最終更新日）
- `docs/image-saver-docs.html` — 技術仕様書（バージョン・更新日・変更履歴タイムライン）
- `docs/image-saver-help.html` — ヘルプ（最終更新日・機能説明）
- `docs/image-saver-setup.html` — 導入手順（最終更新日）

### 禁止事項
- 過去セッションの実績（「前回は home.html だけだった」等）を今回のスコープ判断の根拠にしない
- 「簡単な作業」と判断して grep 調査を省略しない
- 更新前の確認なしに編集しない

### 更新後
- `git commit & push`（リモート名：`portfolio19861119`）
- 静的 HTML のため、プレビューパネルは使用しない（devサーバー未起動のため表示が崩れる）
- ライブサイトへの反映を確認方法として案内する
